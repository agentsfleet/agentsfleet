Credentials = the secret vault (not just model keys)

  • Three kinds, one vault. The strip up top spells it out: model providers, integrations, and custom secrets. Adding a secret is first-class, not buried.
  • Integrations ship later — custom secrets bridge the gap now. GitHub/Zoho/Slack are marked "Coming soon"; today you store their tokens as custom secrets (GITHUB_TOKEN) and fleets use them. When the connector lands, it adopts the secret. So this round ships Model providers + Custom secrets (both real), no half-built connectors.
  • Custom secrets are arbitrary NAME=value entries your SKILL.md reads by name — the "+ Add secret" row is right there.
  • Referenced-by column shows which fleet uses each secret, so you never delete something live by accident.
  • Write-only throughout. Stored as a vault key_name ref (resolved via crypto_store), masked on display, Replace not reveal.
  • Models references this vault — the own-key path picks a model-provider credential from here; it doesn't store keys of its own.
